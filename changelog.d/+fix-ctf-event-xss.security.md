**Stored XSS in CTF event form via scenario names is fixed.** `admin_event_create` and `admin_event_edit` no longer serialize scenario data with `json.dumps()` and pass it through `|safe` in the template. The views now pass a plain Python list and the template embeds it with Django's `json_script` filter, which escapes `<`, `>`, and `&` for safe inline script use. A scenario name containing `</script>` can no longer break out of the script block.
