**Portal `/health` now reflects real dependency probes instead of always returning 200.** The endpoint runs the registered `django-health-check` database, cache, and storage checks and returns 500 when any probe fails. Load-balancer probes still admit past `ALLOWED_HOSTS` via a path-scoped Host-header normalization, and the public response stays coarse (a `working` / `unavailable` token per probe) so dependency failures do not leak DSNs, bucket names, or private hostnames.
