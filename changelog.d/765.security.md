**Defense-in-depth ownership checks added to organizer-scoped CTF challenge services.** `create_challenge`, `update_challenge`, `delete_challenge`, and `list_challenges_for_event` now require an `actor_id` keyword argument and raise `CTFPermissionError` when the actor does not own the event. This is a backstop for the existing view-layer `_check_event_ownership` check, so a future internal caller that bypasses the views still cannot mutate another organizer's event content. Cross-organizer regression tests added for the JSON challenge APIs.
