**Deploy verification now fails loud instead of reporting a false-green deploy.** The Guacamole stabilization wait fails the run on timeout instead of warning and exiting 0 (a broken `guacd` image no longer passes silently), and the engine ECS deploy fails when the task-definition family cannot be described instead of skipping forever on a typo'd family name. A genuine first-ever deploy to a fresh AWS environment can still skip the engine task-family check via the new strict-default `aws_first_deploy` manual-dispatch input. A new `deploy-verification-fail-loud` ADR guard check (ADR-003-R3) keeps the invariant from regressing.
