Range Architecture

Each team range follows the same topology. The NGFW sits inline between the server subnet and endpoint subnet, with a separate untrust interface for internet egress and a management interface for admin access.

To decrypt the built-in Administrator password:

# Windows Server key
aws secretsmanager get-secret-value --secret-id shifter/dev/windows-server-key \
  --query SecretString --output text > ws.pem && chmod 600 ws.pem

# Windows Desktop key
aws secretsmanager get-secret-value --secret-id shifter/dev/windows-desktop-key \
  --query SecretString --output text > wd.pem && chmod 600 wd.pem

# Decrypt (replace INSTANCE_ID)
aws ec2 get-password-data --instance-id INSTANCE_ID --priv-launch-key ws.pem

The victim user accounts (victimadminteamN / victimuserteamN) are created by EC2 userdata at first boot. The victimadmin user is in the Administrators and Remote Desktop Users groups. The victimuser is in Remote Desktop Users only.

Infrastructure Management

Terraform

Team1 is managed separately in platform/terraform/global/tssummit/.

Teams 2-10 use a shared config in platform/terraform/global/tssummit-ranges/ with per-team state:

cd platform/terraform/global/tssummit-ranges

# Switch to a team's state
terraform init -backend-config=backend-teamN.hcl -reconfigure

# Plan/apply for that team
terraform plan -var-file=teamN.tfvars
terraform apply -var-file=teamN.tfvars -parallelism=5

ZTNA Connectors

Deployed via CloudFormation. Stack names: tssummit-ztna-teamN. Credentials stored in temp/tssummit/.env.

Remaining Setup Steps

  1. Configure NGFW interfaces with zones and IPs (via Strata Cloud Manager or manual CLI)
  2. Push security policies to NGFWs (Team1 has allow-any-any; Teams 2-10 need the same or equivalent)
  3. Team1 NGFW needs untrust EIP attached (pending Terraform apply)
  4. Verify ZTNA connectors are registered in Prisma Access
