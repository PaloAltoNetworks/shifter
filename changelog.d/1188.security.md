**CTF HTTP flag validators now pin outbound connections to a pre-validated IP.** Closes a DNS-rebinding window where a hostname could resolve to a public address during validation but to a loopback, private, link-local, or cloud-metadata address at request time. Every IPv4/IPv6 answer in the DNS reply must pass SSRF policy or the request is refused, and the TLS socket is opened to the validated address while SNI, certificate verification, and the `Host:` header retain the original hostname.
