**The post-apply RDS pending-modifications check no longer fails AWS platform deploys with spurious "RDS instance not found" errors.** The `db_instance_id` Terraform outputs for the portal and Guacamole RDS instances were emitting `aws_db_instance.id`, which became the `DbiResourceId` (`db-XXXX`) under the AWS provider v6 bump; the check resolves instances by `DBInstanceIdentifier`. Both outputs now emit `aws_db_instance.identifier`.
