**CI verifies the deploy control plane's gating invariants.** A new
`workflow-gating-tests` Quality job parses `deploy.yml` and the reusable deploy
workflows as data and asserts the gating rules that `actionlint` (syntax only)
and `adr_guard` (narrow ADR checks) cannot infer: every deploy job fails closed
when an upstream is `failure`/`cancelled` (#781), branch/event routing keeps
`workflow_dispatch` on `main` as the only production-apply path and never
deploys from `dev`/`main` (#892), the `portal_image` (app image) and
`shifter_platform` (Terraform) change filters stay split (#913), and no
`pull_request` event can reach a self-hosted apply/deploy job (the
runner-exposure fix). A constrained GitHub Actions `if:` expression evaluator
proves each denied case semantically rather than by substring match. The job
runs on every PR with `contents: read` only — no secrets, no cloud access.
