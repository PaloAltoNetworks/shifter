## New Entities

### MCP Session (Runtime Only - Not Persisted)

The MCP server maintains per-user sessions in memory. These are not database entities.

```
MCPSession (in-memory)
├── session_id (UUID) - Generated by StreamableHTTPServerTransport
├── user_email (string) - From JWT claims
├── range_id (integer) - From database lookup
├── kali_ip (string) - From range record
├── ssh_key (string) - From Secrets Manager (cached)
├── lab_config (LabConfig) - Dynamic config for handlers
├── transport (StreamableHTTPServerTransport)
└── created_at (timestamp)
```

**Lifecycle**:
1. Created on first MCP request from user
2. Cached for session duration (mcp-session-id header)
3. Cleaned up on session termination or timeout

## Validation Rules

### Range Lookup

| Rule | Description |
|------|-------------|
| Active range required | User must have range with status='ready' |
| Single range per user | Only one active range at a time (enforced by provisioner) |
| Kali IP required | Range must have `kali_ip` populated |
| SSH key ARN required | Range must have `kali_ssh_key_secret_arn` populated |

### JWT Claims

| Claim | Validation |
|-------|------------|
| `email` | Required, must be non-empty string |
| `exp` | Must be in future (not expired) |
| `iss` | Must match Cognito user pool issuer |
| `aud` / `client_id` | Must match OpenWebUI app client ID |

## State Transitions

No new state transitions. Existing Range status flow unchanged:

```
pending → provisioning → ready → destroying → destroyed
                      ↘ failed
```

**MCP Integration Points**:
- MCP server reads ranges in `ready` status only
- MCP returns error for non-ready ranges
- Provisioner sets `kali_ssh_key_secret_arn` during `create_kali` step

## Query Patterns

### Primary Query: Range by User Email

**Used by**: MCP server on every request

```sql
SELECT r.*
FROM mission_control_range r
JOIN auth_user u ON r.user_id = u.id
WHERE u.email = $1
  AND r.status = 'ready'
ORDER BY r.created_at DESC
LIMIT 1;
```

**Indexes Needed**:
- `auth_user.email` (likely already indexed)
- `mission_control_range.status` (already indexed per model definition)

### SSH Key Retrieval

**Used by**: MCP server when session initialized

```python
secrets_manager.get_secret_value(SecretId=range.kali_ssh_key_secret_arn)
```

**Caching**: SSH key cached in MCP session for session lifetime (avoid repeated Secrets Manager calls)
